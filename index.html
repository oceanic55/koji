Best Practices for Setting Up Rules in Trae IDE for Document Research

Introduction

Trae IDE, an AI-powered Integrated Development Environment, offers a robust system for setting up rules to customize and optimize workflows, including document research. As of the latest updates (May 31, 2025), Trae supports three types of rules—User Rules, Project Rules, and Agent Rules—to provide precise control over AI behavior and project context. This guide outlines the best practices for configuring these rules specifically for document research, ensuring efficient management, organization, and retrieval of research materials. The information is based on the latest Trae IDE documentation and community insights from X posts.

Section 1: Understanding Rules in Trae IDE

Trae IDE allows users to define rules to guide AI behavior across different scopes:

  • User Rules: Apply globally to all projects, setting default behaviors for the AI across your workspace.
  • Project Rules: Define project-specific guidelines, ideal for tailoring AI responses to the context of a particular research project.
  • Agent Rules: Add custom instructions to individual AI agents, enabling fine-tuned control for specific tasks like document analysis.

These rules can be layered to create a hierarchical structure, ensuring consistent and context-aware AI assistance for document research.

Section 2: Setting Up Rules for Document Research

Document research involves gathering, organizing, and analyzing textual data, often requiring clear structure, consistent formatting, and accurate referencing. Below are the steps to set up rules in Trae IDE to optimize this process.

2.1 Configuring User Rules for General Research Standards

  1. Open Trae IDE and navigate to Settings > User Rules from the main menu.
  2. Create a new User Rules file (e.g., user_rules.md) in a Markdown format to define global research standards.
  3. Add guidelines to enforce consistency across all research projects, such as:
    • Always use Markdown for documentation to ensure compatibility with version control systems like Git.
    • Include a standard citation format (e.g., APA or MLA) for references in research documents.
    • Require a changelog for tracking updates to research documents.
    Example User Rules: 
    # User Rules for Research
- All documentation must be written in Markdown.
- Citations must follow APA format: (Author, Year).
- Maintain a changelog in a separate file (e.g., changelog.md) to track document updates.
- Use clear, concise language without jargon unless necessary for the audience.
  4. Save the rules file in the default user configuration directory (e.g., ~/.trae/user_rules/).
  5. Trae’s AI will apply these rules across all projects unless overridden by Project or Agent Rules.

Note: User Rules are ideal for setting a baseline for documentation practices, ensuring consistency across multiple research projects.

2.2 Configuring Project Rules for Specific Research Projects

  1. Open the project in Trae IDE.
  2. Navigate to Project > Project Rules or create a .trae/rules/ directory in the project folder.
  3. Create a Project Rules file (e.g., project_rules.md) to define guidelines specific to the research project. For document research, include:
    • Naming conventions for research files (e.g., topic_date.md).
    • A requirement to include a table of contents in each document for easy navigation.
    • Specific instructions for organizing references (e.g., store all references in a references.bib file).
    • Guidelines for AI to summarize documents in bullet points before analysis.
    Example Project Rules: 
    # Project Rules for Document Research
- File names: Use format [topic]_[YYYYMMDD].md (e.g., AI_Research_20250601.md).
- All documents must include a table of contents with at least two levels of headings.
- Store references in references.bib using BibTeX format.
- AI must generate a bullet-point summary of each document before analysis.
  4. Save the rules file in the .trae/rules/ directory. Trae will automatically detect and apply these rules to the project.

Note: Project Rules allow customization for specific research goals, such as organizing literature reviews or technical reports.

2.3 Configuring Agent Rules for Document Analysis Tasks

  1. In Trae IDE, open the AI agent panel (e.g., via View > AI Agents).
  2. Select or create an agent dedicated to document research tasks (e.g., “ResearchBot”).
  3. Add Agent Rules in the agent’s configuration (e.g., researchbot_rules.md) to guide specific tasks, such as:
    • Extract key findings from documents and present them in a structured format.
    • Highlight potential biases or gaps in the research material.
    • Cross-reference cited sources with a provided bibliography.
    Example Agent Rules: 
    # Agent Rules for ResearchBot
- Extract key findings as bullet points with a maximum of 5 points per document.
- Flag potential biases or gaps in the document with a brief explanation.
- Verify all citations against references.bib and report missing entries.
- Use concise language and avoid speculative conclusions.
  4. Save the rules in the agent’s configuration. These rules will guide the AI’s behavior for specific research tasks.

Note: Agent Rules are particularly useful for fine-tuning AI responses during document analysis, ensuring accurate and relevant outputs.

Section 3: Best Practices for Document Research Rules

Based on Trae IDE’s documentation, X posts, and general documentation best practices, the following are recommended rules for document research:

  • Clarity and Simplicity: Write rules in clear, concise language to ensure the AI understands and follows them. Avoid ambiguous terms and use examples where possible.
    • [](https://scribehow.com/library/technical-documentation-best-practices)
  • Version Control Integration: Store rules and research documents in a Git repository for version control. Use Markdown files for compatibility with Trae’s source control features.
    • [](https://daily.dev/blog/documentation-version-control-best-practices-2024)
  • Structured Documentation: Require a consistent structure (e.g., table of contents, headings, and references) to make documents easy to navigate and understand.
    • [](https://swimm.io/blog/the-four-main-documentation-strategies-explained)
  • Incremental Updates: Update rules incrementally as research needs evolve. Regularly audit rules to remove outdated or redundant instructions.
    • [](https://www.software.ac.uk/blog/what-are-best-practices-research-software-documentation)
  • Changelog Maintenance: Maintain a changelog (e.g., changelog.md) to track document updates and AI task completions. This helps Trae’s AI maintain context across research phases.
    • [](https://www.reddit.com/r/TraeIDE/comments/1ilaeqh/am_i_missing_something_rules_file_support/)
  • Contextual Awareness: Include a comprehensive requirements document in the project’s context to guide the AI. For example, specify research phases and objectives to ensure relevant suggestions.
    • [](https://www.chatprd.ai/resources/PRD-for-Cursor)
  • Feedback Integration: Set rules to prompt the AI to request feedback on generated summaries or analyses, ensuring accuracy and relevance.
    • [](https://jhudatascience.org/Documentation_and_Usability/what-does-good-documentation-look-like.html)

Section 4: Example Workflow for Document Research

Here’s an example of how to use layered rules for a literature review project:

  1. User Rules: Mandate Markdown for all documents, APA citations, and a changelog for tracking updates.
  2. Project Rules: Require files to be named literature_[topic]_[YYYYMMDD].md, include a table of contents, and store references in references.bib.
  3. Agent Rules: Instruct the ResearchBot agent to summarize each article in 3–5 bullet points, flag potential biases, and verify citations.
  4. Execution: Open a research document, use the ResearchBot agent to generate summaries, and commit changes to GitHub with a descriptive commit message (e.g., “Added summary for AI ethics article”).
  5. Review: Periodically audit the changelog and update rules based on research progress or new requirements.

Section 5: Troubleshooting

  • AI Ignoring Rules: Ensure rules files are saved in the correct directories (~/.trae/user_rules/ for User Rules, .trae/rules/ for Project Rules). Check for syntax errors in Markdown files.
  • Inconsistent AI Outputs: Refine Agent Rules to be more specific and include examples. Increase context by referencing requirements documents or changelogs.
    • [](https://www.reddit.com/r/TraeIDE/comments/1ilaeqh/am_i_missing_something_rules_file_support/)
  • Version Control Conflicts: Pull the latest changes from the GitHub repository before committing to avoid conflicts. Use Trae’s Source Control panel to resolve conflicts if they arise.

Setting up rules in Trae IDE for document research enhances efficiency and consistency by leveraging User, Project, and Agent Rules. By following best practices such as maintaining clear, structured, and version-controlled documentation, researchers can optimize Trae’s AI capabilities for literature reviews, technical reports, and other research tasks.
